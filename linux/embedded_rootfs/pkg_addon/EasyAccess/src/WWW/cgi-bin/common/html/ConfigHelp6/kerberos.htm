The Add New Domain page enables you to create a new authentication domain.

Type topic text here.

 

NOTE: Of all types of authentication, Kerberos authentication is the most error prone. If you are unable to authenticate using Kerberos, please read the troubleshooting procedure at the end of this section.

 

In the Add New Domain window:

 

   1.     Select Kerberos from the Authentication Type menu. The Kerberos   

           configuration fields will be displayed.

 

   2.     Enter a descriptive name for the authentication domain in the Domain Name

           field. Users will select this domain when they log into the SSL VPN portal. It

           can be the value as the Server Address field or the Kerberos Domain

           field depending on your network configuration.

 

   3.     Enter the IP address or fully qualified domain name of the Kerberos server in

           the Server Address field.

 

   4.     Enter the Kerberos domain name in the Kerberos Domain field.

 

   5.     Enter the name of the layout in the Portal Layout Name field. The default

           layout is EasyAccess. Additional layouts may be defined in the Portal Layouts

           page.

           NOTE: If you select a portal other than EasyAccess, then the domain will not

           be displayed on the default login page. Users will either need to login as

           https://[IP/Domain Name/portal/Portal Name].

 

   6.     Click Submit to update the configuration. Once the domain has been added,

           the domain will be added to the Domains table.

 

 Users that have been defined in the Kerberos database can log into the EasyAccess portal by entering their Kerberos user name and password and selecting the new Kerberos authentication domain from the Domain menu in the EasyAccess Authentication window.

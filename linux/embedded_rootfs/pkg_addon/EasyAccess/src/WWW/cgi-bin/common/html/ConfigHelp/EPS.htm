EPS-HELP

Overview

 

End Point Security works on client/server model. The administrator of the EasyAccess SSL VPN System can configure a set of policies at the server (through the Web UI), that all client computers that connect to the corporate network through SSL VPN should adhere to. EPS policies can be defined for global or a group access. These policies include the Operating Systems / Browsers that the clients are allowed to run and the Service packs/patches that should be present. They may also include the presence and status of specific versions and updates of Firewall software, Antivirus software.  Additional policies may be configured to check the presence or absence of registry key-value pairs.

 

The client program is installed on or downloaded to every computer (endpoint) that connects to the corporate network through the EasyAccess SSL VPN system. The client program collects information and sends this information to the server. The server validates the policies configured in the server against the information provided by the EndPoint Client module that executes on the client computer. Depending on the result of the policy validation the server grants/denies access to the corporate network for the client computer.

 

A compliance policy should check for any one of the service packs (if it defined) and set of OS patches (if it defined) and any one of the browser version (if it defined) and any one of the antivirus policy (if it defined) and any one of the firewall policy (if it defined). It also includes the check for set of Registry key-value pairs (if it defined).

 

Note: To enable EPS Policy, check the checkbox 'To Enable EPS' in the Access > Portal Layout configuration Page.

 

EPS POLICY

 

To view the EasyAccess EPS Policy, click the EPS option in the left navigation menu, then click the EPS Policy option.

 

Add EPS POLICY

 

To create a new EPS Policy, click Add EPS Policy. An EPS Policy window will be displayed.

 

1. Enter the EPS Policy Name in the EPS Policy Name field; select the Operating System and Service Pack from

    the dropdown list and check or un-check UptoDate option that should be present in the EPS Client to logging

    in to  EasyAccess and Click Submit.

 

To include Patch Policy for the EPS Policy, click Add Patch Policy. An Add Patch Policy window will be displayed.

 

1. Select the various Patch List from the multiple select combo box and click Submit.

2. The various Patch List will be added to the EPS Policy.

 

Note: All OS Patches defined in the EPS Policy should exist in the EPS Client.

 

To include Browser Policy for the EPS Policy, click Add Browser Policy. An Add Browser Policy window will be displayed.

 

1. Select the various Browser List from the multiple select combo box and click Submit.

2. The various Browser List will be added to the EPS Policy.

 

Note: EPS Client Browser version should match against any of the browser defined in the EPS Policy

 

To include RegistryKey Policy for the EPS Policy, click Add RegistryKey Policy. An Add RegistryKey Policy window will be displayed.

 

1. Select the various RegistryKey List from the multiple select combo box and click Submit.

2. The various RegistryKey List will be added to the EPS Policy.

 

Note: All OS Registries defined in the EPS Policy should exist in the EPS Client.

 

To include Antivirus Policy for the EPS Policy, click Add Antivirus Policy. An Add Antivirus Policy window will be displayed.

 

1. Select the various Antivirus List from the multiple select combo box and click Submit.

2. The various Antivirus List will be added to the EPS Policy.

 

Note: EPS Client Antivirus should match against any of the Antivirus policies defined in the EPS Policy.

 

To include Firewall Policy for the EPS Policy, click Add Firewall Policy. An Add Firewall Policy window will be displayed.  

 

1. Select the various Firewall List from the multiple select combo box and click Submit.

2. The various Firewall List will be added to the EPS Policy.

 

Note: EPS Client Firewall should match against any of the Firewall policies defined in the EPS Policy.

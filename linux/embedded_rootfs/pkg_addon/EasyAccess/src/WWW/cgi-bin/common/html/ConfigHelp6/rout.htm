Routes

From the Routes window, an administrator can define the default network route and add additional static routes and local area subnets. The static routes are optional; the default network route is required for Internet access.

 

All currently configured routes are displayed in the Current Routes table.

 

To configure the default route:

 

  1. In the Add Default Route table, enter the IP address of the router or default gateway of the network in the Default Gateway Address field. The default gateway address is same gateway address used by local area network computers to connect to the Internet and it may be the address of a network firewall. IPv6 address is also supported.
     

  2. Select the Ethernet interface that should be used to connect to the default gateway address from the Interface menu.
     

  3. Click Submit to update the configuration. Once the default route has been added, the route will be displayed in the Current Routes or Current IPv6 Routes table.

 

To configure a static route:

 

  1. Enter the destination network address of the static route in the Destination Network field. The destination network address is an IP address in the remote network subnet. It can also be an IPv6 address.

    NOTE:     The destination network address may be a valid IP address or it may be a subnet address that ends in “.0”, such as “192.168.0.0”.
     

  2. Enter the subnet mask, if destination network is IPv4 or enter IPv6 prefix, if the destination network is IPv6 in Subnet Mask / IPv6 Prefix field. Valid Ipv6 prefix can range from 0 - 128.
     

  3. Enter the IP address of your router in the Gateway Address field. The gateway address should share the same subnet or prefix with one of the network interfaces configured on the EasyAccess device.

    For example, if the eth0 interface address is “10.0.0.100” and the subnet mask is “255.255.255.0”, then a router connected through the eth0 interface should be “10.0.0.X”. For IPv6, if eth0 interface is “3ffe:f101:0:f101::61”, then a router connected through the eth0 interface should be “3ffe:f101:0:f101::X”.
     

  4. Select the Ethernet interface that should be used to connect to the gateway address from the Interface menu.
     

  5. Click Submit to update the configuration. Once the domain has been added, the domain will be added to the Current Domains table.

 

NOTE: To add additional local subnets that can directly connect to the EasyAccess device, define the Destination Network address, the Subnet Mask, and the Interface, but leave the Gateway Address field blank.

Virtual Transport Installation

To install Virtual Transport and connect to the SSL VPN gateway, perform the following instructions:

 

  1. Click the Connect via Virtual Transport icon to open a connection to the SSL VPN gateway. The Virtual Passage ActiveX client will download and install on your machine. The connection process will take between 5 to 10 seconds.

 

NOTE: Virtual Transport requires Internet Explorer. ActiveX content must also be supported. If you are running Windows XP SP2, you may see a warning message when you attempt to open a Virtual Transport connection. Right click on the warning message and select Accept to allow the Virtual Transport software to install.

 

When the connection is established, a Virtual Transport icon in the Windows Taskbar Notification Area will appear.

 

Your network administrator should have defined specific TCP applications that you can access using Virtual Transport. To view the server addresses and application port numbers, open the Virtual Transport Properties by selecting the Virtual Transport icon in the Windows Taskbar Notification Area. In the connections table, you should see a list of configured servers that are available. For more information, contact your network administrator.

 

To end the SSL VPN Virtual Passage connection, open the Virtual Transport Properties by selecting the Virtual Transport icon in the Windows Taskbar Notification Area. Then click Disconnect. The Virtual Transport connection will take approximately 10 seconds to terminate.

 

NOTE: SSL VPN users can only create a Virtual Transport to the remote network during an active SSL VPN session. Once a user logs out of the SSL VPN portal or closes the portal window, the Virtual Transport session will terminate. In order to create a new Virtual Transport connection, the user must log back into the SSL VPN portal and relaunch the Create a Virtual Transport icon.

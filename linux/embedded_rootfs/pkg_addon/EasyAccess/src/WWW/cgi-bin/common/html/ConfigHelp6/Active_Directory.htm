Active Directory

To create a domain with Microsoft Windows Active Directory authentication, click Add Domain in the Access >> Domains window. The Add New Domain window will be displayed.

 

NOTE: Of all types of authentication, Active Directory authentication is the most error prone.  If you are unable to authenticate using Active Directory, please read the troubleshooting procedure at the end of this section.

 

In the Add New Domain window:

 

  1. Select Active Directory from the Authentication Type menu. The Active Directory configuration fields will be displayed.
     

  2. Enter a descriptive name for the authentication domain in the EasyAccess Domain Name field. This is the domain name users will select in order to log into the SSL VPN portal. It can be the same value as the Server Address field or the Active Directory Domain field depending on your network configuration.
     

  3. Enter the IP address or host and domain name of the Active Directory server in the Server Address field. You can also enter IPv6 address, if active directory is configured with IPv6 support.
     

  4. Enter the Active Directory domain name in the Active Directory Domain field.
     

  5. Enter the name of the layout in the Portal Layout Name field. The default layout is EasyAccess. Additional layouts may be defined in the Access >> Portal Layouts page.

    NOTE:     If you select a portal layout other than EasyAccess, then the domain will not be displayed on the default login page. Users will either need to login at https://[IP_or_Domain_Name]/portal/[Portal_Name] or at a virtual hostname.
     

  6. Check the Require Kerberos authentication checkbox to force users to authenticate to Active Directory’s integrated Kerberos authentication mechanism.
     

  7. Check the Require client digital certificates checkbox to force users to supply a valid digital certificate before granting access. The CNAME of the client certificate must match the user name that the user supplies to login and the certificate must be generated by a certificate authority (CA) that is trusted by EasyAccess.
     

  8. Check the Require CIFS bookmark to home directory checkbox to add bookmark for server's Home Directory Base Path, which has Home Directories for that specific domain users.

     

  9. Click Submit to update the configuration. Once the domain has been added, the domain will be added to the Current Domains table.

 

Users that have been defined in the Active Directory database can log into the EasyAccess SSL VPN portal by entering their Active Directory user name and password and selecting the new Active Directory authentication domain from the Domain menu in the EasyAccess Authentication window.

 

Active Directory Troubleshooting

  

If your users are unable to connect via Active Directory, verify the following:

 

  1. The time settings between the Active Directory server and the EasyAccess SSL VPN gateway must be synchronized. Kerberos authentication, used by Active Directory to authenticate clients, permits a maximum of a 15-minute time difference between the Windows server and the client (the SSL VPN gateway). The easiest way to solve this issue is to configure Network Time Protocol on the General >> Date page and check that the server’s time settings are also correct.
     

  2. Confirm that your Windows server is configured for Active Directory authentication. If you are using Window NT4.0 server, then your server only supports NT Domain authentication. Occasionally, Windows 2000 and 2003 servers are also configured for NT Domain authentication to support legacy Windows clients.

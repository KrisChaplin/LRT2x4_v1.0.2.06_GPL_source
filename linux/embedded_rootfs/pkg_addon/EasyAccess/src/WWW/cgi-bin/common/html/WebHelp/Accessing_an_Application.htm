Accessing an Application Using Terminal Services

Sometimes you may wish to run a single application rather than creating a tunnel to the entire desktop. By just opening a single application, you can improve the performance of the encrypted connection. Your network administrator may have defined several applications such as Microsoft Word and Microsoft Excel. To launch these applications, click on the desired application icon in the Terminal Services Applications table.

 

NOTE: If you are accessing the Terminal Services 5.0 ActiveX from Windows XP SP2, please read the special instructions at the end of this chapter.

 

Depending on how your network administrator has configured the SSL VPN portal, each Terminal Services application may be mapped to a specific Terminal Server. If this is the case, then once you click an application, a Java or ActiveX window will open and within several seconds the Terminal Services application will be displayed.

 

If a server has not been configured, then a browser window prompting you to define the server name or address will be displayed.

 

    1. Enter the domain name, host name or IP address of a machine running terminal services in the Address field.
       

    2. Click Submit.

 

If the SSL VPN gateway is able to successfully connect to the specified server and open the application, the application will be displayed in a new window.

 

Do not close the window that opened the application window; closing this window will close the session to the application.

 

If you see an error page that displays “Error: You are not authorized to access this server using this SSL-VPN service” then you do not have permission to access this host using Terminal Services. Contact your EasyAccess administrator for more information.

 

If the window displays an error message, then the server may be unavailable, it may not be correctly configured to accept terminal services or the application may be saved to a different location on the remote machine.

 

Terminal Services 5.0 with Windows XP Professional SP2

 

If you are using Windows XP Professional with Service Pack 2, you will need to download and install an update for Service Pack 2 that provides support for alternative loopback addresses such as 127.0.02.

 

If you try to connect to a Terminal Server from Windows XP SP2 and you see an error stating that the server cannot connect, but the Java-based Terminal Services client works fine, then you need to install the update patch.

 

To download the patch, go to:

 

http://www.microsoft.com/downloads/details.aspx?FamilyID=17d997d2-5034-4bbb-b74d-ad8430a1f7c8&DisplayLang=en

 

Download the patch. After you download it, you may need to restart Internet Explorer or reboot your machine before you can access the application.

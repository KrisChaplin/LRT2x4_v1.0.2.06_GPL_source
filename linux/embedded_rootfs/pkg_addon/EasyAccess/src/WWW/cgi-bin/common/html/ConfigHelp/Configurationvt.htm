Virtual Transport Configuration

To view the Virtual Transport Configuration window, click the Virtual Transport option in the left navigation menu.

Configure Network Resources Available to Users

Virtual Transport detects and reroutes data sent by the remote client to the EasyAccess gateway. To configure Virtual Transport, you must define the internal host machines and TCP applications available to remote users.

 

  1. Enter the IP address of an internal server or host computer in the IP Address field.
     

  2. Enter the TCP port number of the application to be tunneled in the TCP Port field. Please see http://www.iana.org/assignments/port-numbers for a list of registered port numbers. See below for a list of common TCP applications and port numbers.
     

  3. Click Add Application to submit the application information. The IP address and port number submitted should appear in the Configured Applications for Virtual Transport table.

Configure Host Name Resolution

Once all the server and port information has been configured, remote users will be able to access private network servers using Virtual Transport. However, users will need to remember complicated IP addresses of network servers. To improve users' experience, the EasyAccess administrator can also specify name to IP address resolution. Once configured, users will be able to access TCP applications at familiar addresses such as "mail.mycompany.com" or "ftp.mycompany.com".

 

To add a host name for client name resolution:

 

  1. Enter an IP address in the Local Server IP Address field. The address should already be defined in the Configured Applications for Virtual Transport table.
     

  2. Enter a domain name of the internal server in the Fully Qualified Domain Name field.
     

  3. Click Add Host to submit the host-to-name mapping. The IP address and domain name should appear in the Configured Host Names for Virtual Transport Clients table.
     

Now, remote users will be able to securely access network applications once they have logged into the SSL VPN portal and launched Virtual Transport.

LDAP Attributes

The LDAP authentication domain enables the EasyAccess administrator to define LDAP attributes that limit which users may log in to the portal or determine which resources and bookmarks specific LDAP groups will view.

 

Once an LDAP authentication domain is created, a default LDAP group will be created with the same name as the domain name. Although additional groups may be added or deleted from this domain, the default LDAP group may not be deleted.

 

For an LDAP group, you may define LDAP attributes. For example, you can specify that users in the LDAP group must be members of a specific group or organizational unit defined on the LDAP server. Or you can enter an individual LDAP distinguished name. If you define multiple LDAP attributes, then ALL attributes must be met by users.

 

If no attributes are defined, then any user authorized by the LDAP server can be a member of the group.

 

If multiple groups are defined and a user meets all the LDAP attributes for both groups, then the user will be considered part of the group with the most LDAP attributes defined. If the matching LDAP groups have an equal number of attributes, then the user will be considered a member of the group based on the alphabetical order of the groups.

 

NOTE: If a user is manually added to a LDAP group, then the user setting takes precedence over LDAP attributes. For example, the LDAP attribute objectClass="user" is defined for group Group1 and the LDAP attribute memberOf="CN=WINS Users,CN=Users,DC=menlologic,DC=net" is defined for Group2.

 

If user Jane is classified by an LDAP server as a member of the user object class, but is NOT a member of the WINS Users group, Jane should be in classified as a member of EasyAccess Group1.

 

But if the administrator adds Jane to EasyAccess Group2, then the LDAP attributes will be ignored and Jane will be considered a member of Group2.

 

Sample LDAP Attributes

 

You may enter up to 4 LDAP attributes per group. The following are some example LDAP attributes of Active Directory LDAP users:

 

name="Administrator"

memberOf="CN=Terminal Server Computers,CN=Users,DC=menlologic,DC=net"

objectClass="user"

msNPAllowDialin="FALSE"

 

NOTE: If multiple attributes are defined, all attributes must be met by LDAP users.

 

If an LDAP user fails to meet the LDAP attributes for all LDAP groups configured on EasyAccess, then the user will not be able to log into the portal. So the LDAP attributes feature not only allows the administrator to create individual rules based on the LDAP group or organization, it also allows the administrator to only allow certain LDAP users to log into the portal.

Group Access Policies

To edit group policies, click the group name in the Users and Groups window. The Edit Group Settings window will be displayed.

Group Policies and Bookmarks for Active Directory

Active Directory authentication servers support a group and user structure that can be queried when an active directory user logs in. This means that you can create policies and bookmarks for Active Directory users at the group level, without needing to define Active Directory users in EasyAccess. When a user logs in, if there is no corresponding user name configured in EasyAccess, then EasyAccess will query the Active Directory server for the list of groups that the user belongs to. If any of the same groups are defined in the EasyAccess, then policies and bookmarks for the first Windows Active Directory group that matches a group configured in EasyAccess will be applied to the user.

 

So once you create an Active Directory domain in EasyAccess, you can add groups that correspond with groups on your Active Directory server.

 

If the Active Directory user is configured in EasyAccess, then EasyAccess will ignore the group information provided by Active Directory and instead implement policies and bookmarks based on the user settings and the settings of the group to which the user belongs.

 

Because other authentication services do not have the same hierarchal structure and group definitions as Active Directory, if you want to apply specific policies or bookmarks to a group of RADIUS, NT, or LDAP users, you must add each user on the Users and Groups page.

 

Edit Group Policies

 

With group access policies, all traffic is allowed by default. Additional allow and deny policies may be created by destination address or address range and by service type.

 

The most specific policy will take precedence over less specific policies. For example, a policy that applies to only one IP address will have priority over a policy that applies to a range of IP addresses. If there are two policies that apply to a single IP address, then a policy for a specific service (for example RDP) will take precedence over a policy that applies to all services.

 

NOTE: User policies take precedence over all group policies and group policies take precedence over all global policies, regardless of the policy definition (A user policy that allows access to all IP addresses will take precedence over a group policy that denies access to a single IP address). Refer to Access Policy Hierarchy for more information.

Define Group Access Policies:

  1. In the Apply Policy To menu, select whether the policy will be applied to a predefined network resource, an individual host, a range of addresses or all addresses.
     

  2. Define a name for the policy in the Policy Name field.

    NOTE:     EasyAccess policies apply to the destination address(es) of the SSL VPN connection, not the source address. You cannot permit or block a specific IP address on the Internet from authenticating to the SSL VPN gateway through the policy engine. That type of policy would need to be defined by a firewall rule.
     

  3. If your policy applies to a predefined network resource, select the name of the resource from the Defined Resource menu. For information about creating network resources, refer to Network Resource configuration.

    -   If your policy applies to a specific host, enter the IP address of the local host machine in the IP Address field. It can also be an Ipv6 address.

    -   If your policy applies to a range of addresses, enter the beginning of the IP address range in the Address Range Begin field and the end of the IP address range in the Address Range End field. It can also be an IPv6 address range.
     

  4. Select the service type in the Service menu. If you are applying a policy to a network resource, the service type is defined in the network resource.
     

  5. Select PERMIT or DENY from the Status menu to either permit or deny SSL VPN connections for the specified service and host machine.
     

  6. Click Submit to update the configuration.

 

Once the configuration has been updated, the new group policy will be displayed in the Edit Group Settings window.

 

The group policies will be displayed in the Current Group Policies table in the order of priority, from the highest priority policy to the lowest priority policy.

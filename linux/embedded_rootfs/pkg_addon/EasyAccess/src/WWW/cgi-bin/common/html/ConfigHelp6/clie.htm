Virtual Passage Client Routes

To view the Virtual Passage Client Addresses window, click the Virtual Passage option in the left navigation menu.

Define Client Routes

When remote users connect using the Virtual Passage SSL VPN client, they will receive dynamically assigned IP and DNS settings for their virtual network interfaces. You may define the IP address range to assign to incoming Virtual Passage clients on the Client Addresses window.
 

There are several different scenarios you can use to set up SSL VPN client addresses and routes. The following is a simple network setup.

 

The Virtual Passage SSL VPN Client is a PPP over SSL client. When remote users connect using Virtual Passage, a virtual network interface will be created with IP settings dynamically assigned by the SSL VPN gateway. In addition, DNS and WINS server settings are also assigned by the SSL VPN gateway. DNS and WINS settings allow the Virtual Passage client to contact machines on the corporate network by host name or domain name. The DNS and WINS settings assigned to the Virtual Passage client are configured on the Network >> DNS Settings page.

 

The Virtual Passage client is provided a point-to-point connection and uses proxy ARP requests to locate machines on the remote network.

 

Because the connection is a point-to-point connection, the addresses on the local network and the remote network can overlap. However, there are a few limitations:

 

  • The Virtual Passage client cannot contact a server on the corporate network if Virtual Passage client’s Ethernet interface shares the same IP address as the server or the SSL VPN gateway (i.e., if your laptop has a physical interface address of 10.0.0.45, then you won’t be able to contact a server on the remote network that also has the IP address 10.0.0.45).

  • In addition, you do not want the virtual (PPP) interface address of the Virtual Passage client to conflict with addresses on the corporate network. Therefore, configure an IP address range that does not directly overlap with addresses on your local network. So, if 192.168.0.1 – 192.168.0.100 are currently assigned to machines on your local network, then start the client address range at 192.168.0.101 or choose an entirely different subnet altogether.

 

If you choose a different subnet for the Virtual Passage client range then the subnet used by the corporate network, then you must:

 

  1. Add a client route to configure the Virtual Passage client to connect to the corporate network using the Virtual Passage tunnel.
     

  2. Create a static route on the corporate network’s firewall to forward traffic intended for Virtual Passage client range to the EasyAccess SSL VPN gateway.

 

Once you have determined the address range you will assign to Virtual Passage clients, then define the address range in the EasyAccess administrative interface.

 

Configure the Client IP Address Range

 

  1. Enter the first IP address of the IP address range in the Client Address Range Begin field.

  2. Enter the last IP address of the IP address range in the Client Address Range End field.

  3. Check Force Ipv6 support checkbox to force IPv6 support for VP, when both IPv4 and IPv6 is available in the client.

  4. Enter the first IPv6 address of the IPv6 address range in the Client IPv6 Address Range Begin field, for networks configured with IPv6 support.

  5. Enter the last IPv6 address of the IPv6 address range in the Client IPv6 Address Range End field, for networks configured with IPv6 support.

  6. Click Submit to update the configuration.

 

Restart the EasyAccess software if Virtual Passage clients are actively connected to force these clients to obtain a new virtual IP address.

 

Now, Virtual Passage clients will be able to connect to the SSL VPN gateway and receive a dynamic IP address in the client address range.

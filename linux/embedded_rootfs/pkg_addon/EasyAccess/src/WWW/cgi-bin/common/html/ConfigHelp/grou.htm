Groups

To view the EasyAccess Users and Groups window, click the Access option in the left navigation menu, then click the Users and Groups option.

Add a New Group

To create a new group, click Add Group. An Add Group window will be displayed.
 

  1. Enter a descriptive name for the group in the Group Name field.
     

  2. Select the appropriate domain in the Domain menu. The domain will determine the authentication method for the group.
     

  3. Click Submit to update the configuration. Once the group has been added, the new group will be added to the Users and Groups window.

 

Current Groups

 

All of the configured groups are displayed in the Users and Groups window. The groups are listed in alphabetical order.

 

Delete a Group

 

To delete a group, click the name of the group that you wish to remove in the Users and Groups table. The Edit Group Settings window will be displayed. In the Edit Group Settings window, click Delete Group. The Users and Groups window will be displayed and the deleted group will no longer appear in the list of defined groups.

NOTE: A group cannot be deleted if users have been added to the group or if the group is the default group created for an authentication domain.
 

To delete a group that is the default group for an authentication domain, delete the corresponding domain (you cannot delete the group in the Edit Group Settings window).
 

If the group is not the default group for an authentication domain, first delete all users in the group. Then you should be able to delete the group on the Edit Group Settings page.

 

 

Edit a Group

 

To edit a group, click the name of the group in the Users and Groups table. The Edit Group Settings window will be displayed.

 

The general group information, including the Group Name, Domain Name, Inactivity Timeout and Connection Limit will be displayed. The Group Name and Domain Name are not configurable.

 

To set the inactivity timeout for users in this group:

 

  1. Enter the number of minutes of inactivity to allow in the Inactivity Timeout field.
     

  2. Click Submit to save the configuration changes.

 

The inactivity timeout can be set at the user, group and global level. Set the timeout as 0 in the user and group configuration to use the global timeout setting. If multiple timeout settings are configured, the user timeout setting will take precedence over the group timeout and the group timeout will take precedence over the global timeout.

 

The maximum timeout setting is 2^32 or over 100,000 minutes, although setting the timeout to 0 on the Edit Global Settings page disables the inactivity timeout (if 0 is also configured as the inactivity timeout for the user and group).

 

To set the connection limit for users in this group:

 

     1.  Enter the number of connections to be allowed in the Connection Limit field.

 

     2.  Click Submit to save the configuration changes.

 

Note: Set the Connection Limit to 0 to use the Global Connection Limit settings.

 

The connection limit can be set at the user, group and global level. Set the connection limit as 0 in the user and group configuration to use the global connection limit settings. If multiple connection limits are configured, the user connection limit will take precedence over the group connection limit and the group connection limit will take precedence over the global connection limit.

 

Setting the connection limit to 0 on the Edit Global Settings page disables the connection limit (if 0 is also configured as the connection limit for the user and group).

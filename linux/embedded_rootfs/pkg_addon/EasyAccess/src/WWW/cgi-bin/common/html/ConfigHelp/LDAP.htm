LDAP

To create a domain that requires LDAP authentication, click Add Domain in the Access >> Domains window. The Add New Domain window will be displayed.

 

In the Add New Domain window:

 

  1. Select LDAP from the Authentication Type menu. The LDAP configuration fields will be displayed.
     

  2. Enter a descriptive name for the authentication domain in the EasyAccess Domain Name field. This is the domain name users will select in order to log into the SSL VPN portal. It can be the same value as the Server Address field.
     

  3. Enter the IP address or domain name of the server in the Server Address field.
     

  4. Enter the search base for LDAP queries in the LDAP Base DN field. An example of a search base string is CN=Users,DC=yourdomain,DC=com.

    NOTE:     Currently, the EasyAccess software cannot accept spaces within search base strings. Do not enter any spaces in the LDAP Base DN field.
     

  5. Enter the name of the layout in the Portal Layout Name field. The default layout is EasyAccess. Additional layouts may be defined in the Access >> Portal Layouts page.
     

  6. Check the Require client digital certificates checkbox to force users to supply a valid digital certificate before granting access. The CNAME of the client certificate must match the user name that the user supplies to login and the certificate must be generated by a certificate authority (CA) that is trusted by EasyAccess.
     

  7. Click Submit to update the configuration. Once the domain has been added, the domain will be added to the Current Domains table.

 

Users that have been defined in the LDAP database can log into the EasyAccess SSL VPN portal by entering their LDAP account name and password and selecting the new LDAP authentication domain from the Domain menu in the EasyAccess Authentication window.

Terminal Services Desktop Access

From the My Desktop window, you may use Terminal Services or VNC to access a remote desktop, application, or directory.

 

Remote Desktop Access with Terminal Services

 

To open a connection to a Windows XP desktop or 2000/2003 Terminal Server, click on the Terminal Services icon in the Remote Desktop Access table.

 

    1. Enter the domain name, host name or IP address of the Windows Terminal Server in the Address field.
       

    2. Click Submit.

 

If the SSL VPN gateway is able to successfully connect to the specified Terminal Server, the remote desktop will be displayed.

 

Do not close the window that opened the desktop window; closing this window will close the session to the remote desktop.

 

The EasyAccess Terminal Services client uses Single Sign-On (SSO) authentication, initially supplying the user name and password of the SSL VPN portal to the Windows Terminal Server. If this user name and password is not accepted, the Terminal Services client will prompt you to enter a user name and password.

 

Once you have logged in, you will be able to open files and applications, make file changes and save them on the remote machine.

 

If you see an error page that displays “Error: You are not authorized to access this server using this SSL-VPN service” then you do not have permission to access this host using Terminal Services. Contact your EasyAccess administrator for more information.

 

If the window displays an error message, then the server may be unavailable or it may not be correctly configured to accept Terminal Services connections.  

 

Terminal Services 5.0 with Windows XP Professional SP2

 

If you are using Windows XP Professional with Service Pack 2, you will need to install an update for Service Pack 2 that provides support for alternative loopback addresses such as 127.0.02.  

 

If you try to connect to a Terminal Server from Windows XP SP2 and you see an error stating that the server cannot connect, then you may need to install the update patch.

 

To download the patch, go to:

 

http://www.microsoft.com/downloads/details.aspx?FamilyID=17d997d2-5034-4bbb-b74d-ad8430a1f7c8&DisplayLang=en

 

Download the patch. After you download and install it, you may need to restart Internet Explorer or reboot your machine before you can access the application.

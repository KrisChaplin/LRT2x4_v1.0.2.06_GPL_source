HTTPS Proxy

To access a HTTPS server on your private Intranet site, click on the HTTPS icon in the My Services table. A new browser window will be displayed.

 

  1. Enter the domain name, host name or IP address of the HTTPS server in the Address field.

 

If you clicked on the web application icon to access an Intranet web site, type the name of an intranet site, for example secureintranet.mycompany.com. You can enter the same IP address, domain name or host name that you would use to access the site if you were located on the internal network.

 

NOTE: You do not need to enter the prefix http:// or https://.

 

If you wish to access a site hosted on a port other than port 443 (HTTPS), enter :[PORT NUMBER] after the name or address in the Address field. For example, enter secureintranet.mycompany.com:8080.

 

  2.  Click Submit.

 

If the SSL VPN gateway is able to successfully connect to the specified server, an Intranet HTTPS site will be displayed. The top of the window will display the name or IP address of the server that you are accessing.

 

NOTE: Java applets and ActiveX programs that are downloaded to the browser and then connect back to a server on the corporate network will not function correctly. For client to server connections, use the Virtual Passage SSL VPN client.

 

If you see an error page that displays “Error: You are not authorized to access this server using this SSL-VPN service” then you do not have permission to access this host using the EasyAccess Web proxy application. Contact your EasyAccess administrator for more information.

 

If the window displays an error message, then the HTTPS server may be unavailable.

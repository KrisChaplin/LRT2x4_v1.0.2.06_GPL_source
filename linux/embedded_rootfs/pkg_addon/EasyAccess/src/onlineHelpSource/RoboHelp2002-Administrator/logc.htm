Log Settings

EasyAccess supports web based logging, syslog logging and email alert messages. In addition, EasyAccess may be configured to email the event log file to the EasyAccess administrator before the log file is cleared. Syslog is an industry-standard logging protocol that records system and networking activity.

The EasyAccess syslog messages are sent in WELF (WebTrends Enhanced Log Format), so most standard firewall and networking reporting products can accept and interpret the EasyAccess log files.

 

The EasyAccess syslog service transmits syslog messages to external syslog server(s) listening on UDP port 514.

To configure log and alert settings, complete the following steps:

     

  1. Enter the IP address or fully qualified domain name of your syslog server in the Primary Syslog Server field. Leave this field blank if you do not require syslog logging.

     

  2. If you have a backup or second syslog server, enter the server’s IP address or domain name in the Secondary Syslog Server field.

     

  3. To receive event log files via email, enter your full email address (username@domain.com) in the Email Event Logs to field. The event log file will be emailed to the specified email address before the event log is cleared. If this field is left blank, log files will not be emailed.

     

  4. To receive alert messages via email, enter your full email address (username@domain.com) or an email pager address in the Email Alerts to field. An email will be sent to the email address specified if an alert event occurs. Define the type of events that will generate alert messages in the Log and Alert Categories table. If this field is left blank, alert messages will not be emailed.

     

  5. To email log files or alert messages, enter the domain name or IP address of your mail server in the Mail Server field. If this field is left blank, log files and alert messages will not be emailed.

     

  6. Enter the email address that log and alert messages will be emailed from in the Mail From Address field.

     

  7. If the mail server requires authentication, enter the user name and password in the Mail User Name and Mail Password fields. NOTE: Only plain text authentication using 'AUTH LOGIN' mechanism is supported.

     

  8. Designate when log files will be cleared and emailed to an administrator in the Send Event Logs field. If the option “When Full” is selected, the event log will be emailed and then cleared from EasyAccess when the log file is full. If “Daily” or “Weekly” options are selected, then the log file will be emailed and deleted on a daily or weekly basis. If “Daily” is selected, the log file will be emailed  at the configured time ranges between 00 hrs and 23 hrs. If "Weekly" is selected, the log file will be emailed at the configured Day and Time. If "Daily" or "Weekly" are chosen, the log file will still be cleared, if the log file is full before the  end of the period. 

     

  9. Click the Clear Log button to delete the current event log. The event log will not be emailed.

     

  10. Define the types of log messages that will be identified as syslog, event log, alert or debug messages in the Log and Alert Categories table. Log categories are organized from most to least critical. If a category is selected for a specific logging service, then that log category and more critical events will be logged.

    For example, if the Error radio button is selected for the Event Log service, then all Emergency, Alert, Critical, and Error events will be stored in the internal EasyAccess log file.

     

  11. Click Submit to update your configuration settings.

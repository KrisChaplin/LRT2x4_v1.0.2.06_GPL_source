Internal User Database

You may create multiple domains that authenticate users with users and passwords stored on the EasyAccess gateway. This is necessary if you wish to display different portal layouts (such as SSL VPN portal pages, themes, etc.) to different users.

 

To create a domain with that authenticates, click Add Domain in the Access >> Domains window. An Add New Domain window will be displayed.

 

In the Add New Domain window:

 

  1. Select EasyAccess Password from the Authentication Type menu.

  2. Enter a descriptive name for the authentication domain in the Domain Name field. This is the domain name users will select in order to log into the SSL VPN portal.

  3. Enter the name of the layout in the Portal Layout Name field. The default layout is EasyAccess. Additional layouts may be defined in the Access >> Portal Layouts page.

  4. Check the Require client digital certificates checkbox to force users to supply a valid digital certificate before granting access. The CNAME of the client certificate must match the user name that the user supplies to login and the certificate must be generated by a certificate authority (CA) that is trusted by EasyAccess.

    NOTE:     If the Require client digital certificates checkbox is checked, then users will not be able to authenticate unless they have a client certificate.

  5. Click Submit to update the configuration.

Once the domain has been added, the domain will be added to the Current Domains table.

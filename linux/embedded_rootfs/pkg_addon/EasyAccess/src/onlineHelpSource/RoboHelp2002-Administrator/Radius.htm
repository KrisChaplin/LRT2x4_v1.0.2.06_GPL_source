Radius

To create a domain with Radius authentication, click Add Domain in the Access >> Domains window. An Add New Domain window will be displayed.

 

In the Add New Domain window:

 

  1. Select Radius from the Authentication Type menu.

  2. Enter a descriptive name for the authentication domain in the Domain Name field. This is the domain name users will select in order to log into the SSL VPN portal.

  3. Enter the IP address of the Primary Radius server in the Primary Radius Server field.

  4. If required by the Radius server, enter an authentication secret in the Primary Secret Password field.

  5. Enter the IP address of the Secondary Radius server in the Secondary Radius Server field.

  6. If required by the Radius server, enter an authentication secret in the Secondary Secret Password field. In case of Primary Radius server connection failure, EasyAccess should connect to Secondary Radius server for authentication.

  7. Enter the name of the layout in the Portal Layout Name field. The default layout is EasyAccess. Additional layouts may be defined in the Access >> Portal Layouts page.

  8. Check the Require client digital certificates checkbox to force users to supply a valid digital certificate before granting access. The CNAME of the client certificate must match the user name that the user supplies to login and the certificate must be generated by a certificate authority (CA) that is trusted by EasyAccess.

  9. Click Submit to update the configuration.

Once the domain has been added, the domain will be added to the Current Domains table.

 

Users that have been defined in the Radius server database can log into the EasyAccess SSL VPN portal by entering their Radius user name and password and selecting the new Radius authentication domain from the Domain menu in the EasyAccess Authentication window.

 

Note: For Radius CHAP Authentication, the user's password should be configured with Reversible Encryption.

Network Resources

To view the EasyAccess Network Resources window, from the Administrative User Interface, click the Access option in the left navigation menu, then click Network Resources.  

 

Network Resources are groups of host names, IP addresses and IP address ranges. By defining resource objects, you can more quickly configure network policies. This is because you will not need to redefine the same set of IP addresses or address ranges when configuring the same access policies for multiple users.

 

Defining Network Resources is optional; smaller organizations can choose to create access policies using individual IP addresses or address ranges rather than predefined Network Resources. But for most organizations, it is recommended to use Network Resources. If server or network configuration changes, you may quickly update the Network Resource rather than needing to update all of the user and group policies.

 

To configure a network policy:

 

  1. Click the Add Resource button. An Add Resource window will be displayed.

  2. In the Add Resource window, type a name for the Network Resource.

  3. Then click Submit. The new Network Resource will be displayed in the Network Resources window.

  4. Click the Resource Name in the table to display the Edit Network Resource window.

  5. In the Add Network Resources table, select either IP Address or IP Address Range and Port Number from the Network Resource menu.

    -  If IP Address was selected, enter an IP address or fully qualified domain name in the IP Address/Name field.
    -  If IP Address Range was selected, enter the beginning IP address of the address range in the IP Range Start field. Enter the end of the address range in the IP Range End field. FTP should support path based policy.

  6. Click Add to add the address or address range to the Network Resource. The new configuration will be displayed in the Defined Resources table.

 

You may define up to 128 addresses or address ranges per Network Resource.

 

To delete a defined resource, click the Delete (X) icon in the Defined Resources table for the resource you wish to delete.

 

To create policies based on these network objects, refer to User, Group and Global policies for instructions on configuring access policies.

Certificates

Generate a Certificate Signing Request (CSR) file or Self-signed Certificate (CRT) and view current SSL certificate settings by clicking General in the left navigation menu and then clicking Certificates.

 

From the Certificates window, you can view the currently loaded certificate, upload a digital certificate and generate a new Certificate Signing Request (CSR) or Self-singed Certificate (CRT).

Upload a new certificate:

  1. In the Import Digital Certificate table, select the Browse button.
     

  2. Locate the zipped digital certificate file on your disk or network drive. Any file name will be accepted, but it must have the “.zip” extension. The zipped file should contain a certificate file named “server.crt” and a certificate key file named “server.key”. If the zipped file does not contain these two files, the zipped file will not be uploaded.

 

Once the certificate has been uploaded, the certificate will be displayed in the Current Certificates table. You may make the new uploaded certificate the certificate used by the EasyAccess software by selecting the Enable radio button for the new certificate and clicking Submit. When the EasyAccess software is restarted it will use the new certificate.

 

NOTE: Most valid certificates created by a Certificate Authority (CA) require a password. Before you enable the certificate and restart the software, be sure to enter the correct certificate password on the View Certificate window.

 

In order to get a valid certificate from a widely accepted Certificate Authority such as Verisign or Thawte, you must generate a Certificate Signing Request for your SSL VPN device.

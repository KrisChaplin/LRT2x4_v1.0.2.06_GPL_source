Global Access Policies

To edit Global Access Policies, click Global Settings in the Access >> Users and Groups window. The Edit Global Settings window will be displayed.

 

To set the inactivity timeout for global settings:

 

       1. Enter the number of minutes of inactivity to allow in the Inactivity Timeout field.

 

       2. Click Submit to save the configuration changes.

 

The inactivity timeout can be set at the user, group and global level. Set the timeout as 0 in the user and group configuration to use the global timeout setting. If multiple timeout settings are configured, the user timeout setting will take precedence over the group timeout and the group timeout will take precedence over the global timeout.

 

The maximum timeout setting is 2^32 or over 100,000 minutes, although setting the timeout to 0 on the Edit Global Settings page disables the inactivity timeout (if 0 is also configured as the inactivity timeout for the user and group).

 

To set the connection limit for global settings:

 

     1.  Enter the number of connections to be allowed in the Connection Limit field.

 

     2.  Click Submit to save the configuration changes.

 

The connection limit can be set at the user, group and global level. Set the connection limit as 0 in the user and group configuration to use the global connection limit settings. If multiple connection limits are configured, the user connection limit

will take precedence over the group connection limit and the group connection limit will take precedence over the global connection limit.

 

Setting the connection limit to 0 on the Edit Global Settings page disables the connection limit (if 0 is also configured as the connection limit for the user and group).

Edit Global Policies

To define global access policies, click Add Policy. An Add Policy window will be displayed.

 

NOTE: User and group access policies will take precedence over global policies.

 

  1. Define a name for the policy in the Policy Name field.
     

  2. Select whether the policy will be applied to a specific host machine, a range of IP addresses or all addresses.

    NOTE:     EasyAccess policies apply to the destination address(es) of the SSL VPN connection, not the source address. So you cannot permit or block a specific IP address on the Internet from authenticating to the SSL VPN gateway through the policy engine. That type of policy would need to be defined by a firewall.

    -      If your policy applies to a specific host, enter the IP address of the local host machine in the IP Address field.It can also be an IPv6 address.
    -  If your policy applies to a range of addresses, enter the beginning of the IP address range in the Address Range Begin field and the end of the IP address range in the Address Range End field. It can also be an IPv6 address range.
     

  3. Select the service type in the Service menu.
     

  4. Select PERMIT or DENY from the Status menu to either permit or deny SSL VPN connections for the specified service and host machine.
     

  5. Click Submit to update the configuration. Once the configuration has been updated, the new policy will be displayed in the Edit Global Settings window.

 

The global policies will be displayed in the Current Global Policies table in the order of priority, from the highest priority policy to the lowest priority policy.

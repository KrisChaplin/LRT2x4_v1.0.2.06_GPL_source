Settings

Configure the EasyAccess software and system settings by clicking General in the left navigation menu and then clicking Settings.

 

The Settings window enables you to save the EasyAccess configuration file, import a saved configuration file, upgrade the EasyAccess software, restore the settings to factory defaults and restart the EasyAccess server.

Export a Backup Configuration File

You may save the EasyAccess configuration settings to a backup file and then import the saved configuration file later.

 

  1. To save a backup version of the EasyAccess configuration, click Export.
     

  2. Choose the location to save the configuration file. The file is named "conf.zip" by default, but it can be renamed.
     

  3. Click Save to save the configuration file.

Import a Configuration File

  1. To import a saved configuration file, click Import.
     

  2. Click Browse to locate a saved configuration zip file. The configuration zip file should contain the smm.conf and fbase.conf files.
     

  3. Select the file and then click Import.
     

  4. Restart the EasyAccess server for the settings changes to take effect.

 

Upgrade the EasyAccess Firmware

 

NOTE: Be sure to export the EasyAccess configuration file before upgrading the firmware in case the software is corrupted or the entire system needs to be reinstalled.

 

  1. To install a new version of the EasyAccess firmware, click Upgrade. The Upgrade firmware window  will be displayed.
     

  2. Click Browse to locate the saved firmware file, EasyAccess-X.X.tar.gz, where X.X indicates the release version.
     

  3. Select the file and then click Upload.
     

  4. Once the file has been uploaded, restart the EasyAccess server for the upgrade to be complete.

 

Install a License Key File

 

The trial version of the EasyAccess software will expire after 15 days. You may import a license key file to convert the trial version to a full version with no expiration date. Contact sales@caviumnetworks.com for information about purchasing an EasyAccess license.

 

1. To import a saved configuration file, click License.
 

2. Click Browse to locate the ap.lic license key file on your hard drive or a shared drive.
 

3. Select the file and then click Upload.

 

The license update will take effect after EasyAccess is restarted. If the license key file is not accepted, then only the administrator will be able to log into the SSL VPN portal; all users will receive an error message when they attempt to login.

Restore the Default Settings

  1. To erase your EasyAccess configuration settings and restore the initial configuration, click Restore.
     

The EasyAccess software will automatically be restarted and all active connections will be dropped.

 

NOTE: The IP address settings are not reset.

Save Configuration Settings to File

If the Save configuration to file after every change checkbox is not enabled, the configuration must be saved manually. This option saves the running configuration to file so that if EasyAccess is rebooted, that the latest configuration will be reloaded.

 

  1. Click the Save Now option to save the running configuration to the smm.conf, tunneld.conf and firebase.conf files.

Save Configuration Settings After Every Change

By default, EasyAccess is configured to save the running configuration to file after every change. This can impact performance and may cause active connections to be dropped depending upon the size of the configuration files. In addition, by disabling this option, if you make a configuration error, you can easily restore your last saved configuration by restarting the EasyAccess software.

 

  1. To disable the option to save settings after every configuration change, check the Save configuration to file after every change checkbox.

 

If you disable this option, be sure to save the configuration file before restarting the EasyAccess software or all configuration changes will be lost.

Encrypt the Configuration File

For security purposes, you can encrypt the configuration files. However, if the configuration files are encrypted, they cannot be edited or reviewed for troubleshooting purposes.

 

  1. To encrypt the configuration files, check the Encrypt configuration file checkbox.

Defining the Layout

Adding an EasyAccess Portal Layout

To add a new portal layout, click the Add Layout button on the Access >> Portal Layout window. The portal layout customization window will be displayed.

 

To define a new EasyAccess Portal Layout:

 

  1. 1. Enter a descriptive name for the portal layout in the Portal Layout Name field.  This name will be part of the path of the SSL VPN portal URL.  

    For example, if your SSL VPN portal is hosted at     https://vpn.company.com, and you created a portal layout named 'sales', then users will be able to access the sub-site at https://vpn.company.com/portal/sales.

    Only alphanumeric characters, hyphen (-), and underscore (_) are accepted for the     Portal Layout Name. If other types of characters or spaces are entered, the layout name will be truncated before the first non-alphanumeric character.

  2. Enter the title for the web browser window in the Portal Site Title field.

  3. If you wish to display a banner message to users before they login to the portal, enter the banner title text in the Banner Title field.  Also enter the banner message text in the Banner Message text area. Then check the Display banner message on login page checkbox to show the banner title and banner message text on the login screen.

  4. If you wish to create separate and distinct portals with different host names, then enter a host and domain name in the Virtual Host Name field. This field is optional.

    If you create a virtual host name, then users can login using a different hostname than your default URL. For example, sales members can access https://sales.company.com instead of the default domain, https://vpn.company.com, that you use for administration. The Portal URL (for example, https://vpn.company.com/portal/sales) will still exist even if you define a Virtual host name. Virtual host names enable administrators to give separate and distinct login URLs to different groups of users.

    Be sure to add an entry in your external DNS server to resolve the virtual hostname and domain name to the external IP address of your SSL VPN gateway.

    Note that if you use virtual host names, you will need to purchase a *.domain SSL certificate or your users may see a “Certificate host name mismatch” warning  when they log into the SSL VPN portal. The certificate hostname mismatch only affects the login page; SSL VPN client applications will not be affected by a hostname mismatch.

    Only alphanumeric characters, hyphen (-) and underscore (_) are accepted for the virtual host name.

  5. Check the Enable HTTP meta tags for cache control checkbox to apply HTTP meta tag cache control directives to this Portal Layout. Cache control directives include:

    <meta http-equiv="pragma" content="no-cache">
    <meta http-equiv="cache-control" content="no-cache">
    <meta http-equiv="cache-control" content="must-revalidate">

    These directives help prevent clients’ browsers from caching SSL VPN portal pages and other web content.

    NOTE:     Enabling HTTP meta tags is strongly recommended for security reasons and to prevent out-of-date web pages, themes and data being stored in users’ web browser cache.

  6. Check the Enable ActiveX web cache cleaner checkbox to load an ActiveX cache control when users login to the SSL VPN portal.

    The web cache cleaner will prompt the user to delete all temporary Internet files, cookies and browser history when the user logs out or closes the web browser window. The ActiveX web cache control will be ignored by web browsers that don’t support ActiveX.

  7. Select the portal pages you wish users to access in the SSL VPN Portal Pages to Display table. Any pages that are not selected will not be visible from the portal navigation menu.

    NOTE:     If you hide portal pages or applications, you should also create SSL VPN access policies that deny access to the corresponding applications. The portal layout only affects the look and feel of the portal, but it does not prevent users from accessing hidden sites.

    The Home Page is an optional starting page for the SSL VPN portal. To configure the Home page, check the Home Page checkbox, then click the Home Page Edit link to edit the contents.

  8. Select the services that users should be able to access in the SSL VPN portal in Services Page - Available Services. Only the corresponding service icons will be visible on the Services page.

  9. Select the desktop clients that users should be able to access in Desktop Page - Available Remote Desktop Clients. Only the corresponding service icons will be visible on the My Desktop page.

  10. Select the desired theme in the Select Theme menu. A preview of the theme will be displayed.

  11. Select the menu layout for the SSL VPN portal in the Select Navigation Layout menu. A preview of the menu layout will be displayed.

  12. Click the Submit button to update the portal layout.

 

The default layout is a centered, framed window. The administrator may optionally configure a top menu layout. The layout change will affect all users and is not customizable by SSL VPN users.

 

The configured theme will be seen by all users who have not configured an individual theme in the SSL VPN Portal Themes window.

 

Apply the portal layout to one or more EasyAccess authentication domains.

 

In addition, if you have selected the option to show the SSL VPN Portal Applications page, then you should also add Terminal Services application icons to display in the Applications page. To do so, scroll down to the bottom of the Portal Layout Customization page.

 

Note: A portal can be selected as Default Portal by enabling the Radio button. Default Portal is allowed only if Domains are associated with it.

 

To add a Terminal Services Application:

 

  1. Enter a description of the application in the Application Description field. This name will be shown beneath the application icon on the SSL VPN portal Applications page.

  2. Enter the path and application name of the Terminal Services application in the Application and Path field.

    NOTE:     To launch a Terminal Services application individually, the Terminal Server must be run in Application mode. In addition, the application must be installed through the Control Panel Add/Remove Programs. For more information, see the Cavium Networks Support Site.

  3. Enter the current working directory path for the Terminal Services application in the Working Directory field

  4. Select an image to show on the Applications page from the Icon Image menu.

  5. Click Add Application to add the new application to the SSL VPN Portal Applications page.

  6. Click the Application Description to edit the application for it's Application path, Working Directory, Icon Image or Host Address.

NT Domain

To create a domain with Microsoft NT Domain authentication, click Add Domain in the Access >> Domains window. The Add New Domain window will be displayed.

 

In the Add New Domain window:

 

  1. Select NT Domain from the Authentication Type menu. The NT Domain configuration fields will be displayed.

  2. Enter a descriptive name for the authentication domain in the EasyAccess Domain Name field. This is the domain name selected by users when they authenticate to the SSL VPN portal. It may be the same value as the NT Domain Name.

  3. Enter the NT authentication domain in the NT Domain Name field. This is the domain name configured on the Windows authentication server for network authentication.

  4. Enter the IP address or host and domain name of the server in the NT Server Address field. You can also enter IPv6 address, if your NT domain server is configured with IPv6 support.

  5. Enter the name of the layout in the Portal Layout Name field. The default layout is EasyAccess. Additional layouts may be defined in the Access >> Portal Layouts page.

  6. Check the Require client digital certificates checkbox to force users to supply a valid digital certificate before granting access. The CNAME of the client certificate must match the user name that the user supplies to login and the certificate must be generated by a certificate authority (CA) that is trusted by EasyAccess.

  7. Check the Require CIFS bookmark to home directory checkbox to add bookmark for server's Home Directory Base Path, which has Home Directories for that specific domain users.

  8. Click Submit to update the configuration.

Once the domain has been added, the domain will be added to the Current Domains table.

Users that have been defined in the NT Domain database can log into the EasyAccess SSL VPN portal by entering their NT Domain name and password and selecting the new NT Domain authentication domain from the Domain menu in the EasyAccess Authentication window.
